# Report Templates

Открывай этот файл, только если пользователь просит подробный отчет по багу,
исправлению или инциденту.

## Короткий шаблон по умолчанию

```md
## Bug
- Симптом:
- Затронутая область:

## Root Cause
- Что именно было сломано:
- Где в коде:

## Fix
- Что изменено:
- Почему это безопасно:

## Validation
- Какие тесты/проверки выполнены:
- Что теперь подтверждено:

## Risk
- Оставшийся риск / follow-up:
```

## Расширенный шаблон для prod/high-risk

```md
## Summary
- Категория:
- Критичность:
- Impact:

## Reproduction
- Как проявлялось:
- При каких условиях:

## Root Cause
- Основная причина:
- Сопутствующие факторы:
- Место в коде:

## Fix
- Что изменено:
- Почему выбран этот вариант:

## Test Gap
- Почему прежние тесты не поймали баг:
- Что добавлено для защиты от регрессии:

## Validation
- Что проверено локально / на staging:
- Что осталось непроверенным:

## Rollback and Monitoring
- Как откатиться:
- За какими метриками/логами следить:

## Follow-up
- Какие действия стоит сделать после срочного фикса:
```

## Правило выбора

- Обычный локальный баг: используй короткий шаблон.
- Продакшен, hotfix, высокий риск, просьба о formal summary: используй
  расширенный шаблон.
