# Bug Categories

Открывай этот файл только если от категории зависит следующий шаг расследования.

## Как выбирать категорию

- Выбирай самую узкую категорию, которая меняет тактику.
- Не пытайся навесить несколько ярлыков, если для расследования достаточно
  одного.
- Если симптомы противоречат друг другу, пометь классификацию как гипотезу и
  проверь ее на коде или воспроизведении.

## Категории

| Категория | Типичные признаки | С чего начать | Как подтвердить |
| --- | --- | --- | --- |
| Логическая ошибка | Неверный результат, неправильная ветка выполнения | Найти функцию и цепочку вызовов | Проверить сломанное бизнес-условие |
| Null/Undefined | NPE, `undefined is not a function`, отсутствующие поля | Найти инициализацию и null-check | Показать путь, где значение не задается |
| Несоответствие типов | Ошибки типов, coercion, несовместимый контракт | Проверить типы, DTO, сериализацию | Найти место расхождения ожиданий |
| Интеграция | Ошибки API, внешних сервисов, очередей, БД | Проверить request/response и retry/timeout | Подтвердить несовпадение контракта |
| Конфигурация | Локально работает, в окружении падает | Проверить env, flags, secrets, версии | Найти различие среды или конфигов |
| Состояние гонки / flaky | Проявляется не всегда, зависит от тайминга | Проверить shared state, async, clock, random | Получить детерминированный сценарий или сильную гипотезу |
| Производительность | Timeout, высокий CPU, медленный запрос | Измерить узкое место, а не гадать | Подтвердить bottleneck цифрами |
| Память / ресурсы | OOM, утечка listeners, рост соединений | Проверить cleanup, cache, lifecycle | Найти ресурс, который не освобождается |
| Безопасность | Инъекция, обход auth, утечка данных | Проверить входную валидацию и доступ | Подтвердить путь обхода или unsafe input |

## Быстрые подсказки

- Регрессия после недавних изменений: смотри `git log`, `git blame`,
  затронутые контракты и предположения.
- Плавающий баг: сначала делай сценарий воспроизведения, только потом меняй
  код.
- Production-only баг: отдельно проверь конфиг, версии, feature flags и данные
  окружения.
- Если у тебя только stack trace, начни от точки падения и двигайся назад по
  входным данным и вызовам.
